# Comparison Test Harness

**Purpose**: Systematically verify behavioral compatibility between old TypeScript Hero and new Mini TypeScript Hero.

**Goal**: 100+ test cases to ensure bulletproof compatibility.

## How It Works

### 1. Test Case Format

Each test case is a TypeScript/JavaScript file with:
- **Input**: Source code with messy imports
- **Expected Output**: What old TypeScript Hero produces

### 2. Comparison Process

```typescript
// For each test case:
const input = readTestCase('test-cases/basic/001-mixed-case-specifiers.ts');

const oldOutput = organizeImportsOld(input, defaultConfig);
const newOutput = organizeImportsNew(input, defaultConfig);

assert.equal(newOutput, oldOutput, "Outputs must match!");
```

### 3. Baseline Generation

```bash
npm run baseline
```

Runs old extension on all test cases and saves outputs as baselines.

### 4. Comparison Testing

```bash
npm test
```

Runs new extension on all test cases and compares against baselines.
